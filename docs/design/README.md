## 設計思想の要点

- **モノレポ構成**: コード共有と一元管理の利便性を重視
- **ドメイン中心設計**: 業務ドメイン単位でモジュール化し、責務を明確化
- **レイヤード + CQRS**: シンプルな CRUD はレイヤード、複雑な読み取りは CQRS で最適化
- **データベース選択**: MySQL（構造化データ）と MongoDB（ドキュメントデータ）を用途に応じて使い分け
- **イベント駆動**: モジュール間連携はドメインイベントで実現し、疎結合を維持

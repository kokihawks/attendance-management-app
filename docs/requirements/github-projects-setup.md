# GitHub Projects バックログ管理設定ガイド

## 概要

本ドキュメントは、GitHub Projectsを使用してプロダクトバックログを管理するための設定手順を説明します。

## 前提条件

- GitHubリポジトリへのアクセス権限
- GitHub Projectsの利用可能なアカウント

---

## 1. GitHub Projectsの作成

### 手順

1. GitHubリポジトリのページにアクセス
2. 上部の「Projects」タブをクリック
3. 「New project」をクリック
4. テンプレートを選択（推奨: **Table** または **Board**）
5. プロジェクト名を入力: `エイ・フォース勤怠管理アプリ - プロダクトバックログ`
6. 「Create project」をクリック

---

## 2. カスタムフィールドの設定

### 推奨フィールド

以下のカスタムフィールドを追加します：

| フィールド名 | タイプ | 説明 |
|------------|--------|------|
| **優先度** | Single select | P0（最優先）/ P1（次優先）/ P2（後回し） |
| **開発フェーズ** | Single select | Phase 2 / Phase 3 / Phase 4 / Phase 5 / Phase 6 |
| **見積り（ポイント）** | Number | ストーリーポイント（1, 2, 3, 5, 8, 13など） |
| **受け入れ基準** | Text | AC-XXX-X形式の受け入れ基準ID |
| **エピック** | Single select | Epic 1〜Epic 8 |
| **機能要件ID** | Text | FR-001〜FR-008 |

### カスタムフィールドの追加手順

1. プロジェクトページで「+」ボタンをクリック
2. 「New field」を選択
3. フィールド名とタイプを設定
4. オプション値を設定（Single selectの場合）

---

## 3. ラベルの設定

### 推奨ラベル

以下のラベルをリポジトリに追加します：

#### 優先度ラベル
- `priority:P0` - 最優先（Must要件）
- `priority:P1` - 次優先
- `priority:P2` - 後回し（Should要件）

#### エピックラベル
- `epic:認証・認可`
- `epic:日報管理`
- `epic:休日登録`
- `epic:勤務表管理`
- `epic:アイデア共有`
- `epic:スケジュール集約`
- `epic:通知`
- `epic:インフラ構築`

#### 開発フェーズラベル
- `phase:Phase2` - 基盤構築
- `phase:Phase3` - MVP - コア機能
- `phase:Phase4` - 拡張機能
- `phase:Phase5` - 高度な機能
- `phase:Phase6` - 最適化・運用準備

#### ステータスラベル
- `status:未着手`
- `status:進行中`
- `status:レビュー中`
- `status:完了`

### ラベルの追加手順

1. リポジトリの「Issues」タブをクリック
2. 「Labels」をクリック
3. 「New label」をクリック
4. ラベル名、色、説明を設定
5. 「Create label」をクリック

---

## 4. マイルストーンの設定

### 推奨マイルストーン

以下のマイルストーンを作成します：

| マイルストーン | 説明 | 目標日 |
|------------|------|--------|
| **M0: 要件定義合意** | Phase 0 完了時 | Phase 0 完了時 |
| **M1: 詳細設計完了** | Phase 1 完了時 | Phase 1 完了時 |
| **M2: 基盤完成** | Phase 2 完了時 | Phase 2 完了時 |
| **M3: MVP リリース** | Phase 3 完了時 | Phase 3 完了時 |
| **M4: 拡張機能リリース** | Phase 4 完了時 | Phase 4 完了時 |
| **M5: 高度機能リリース** | Phase 5 完了時 | Phase 5 完了時 |
| **M6: 本番リリース** | Phase 6 完了時 | Phase 6 完了時 |

### マイルストーンの作成手順

1. リポジトリの「Issues」タブをクリック
2. 「Milestones」をクリック
3. 「New milestone」をクリック
4. マイルストーン名、説明、期限を設定
5. 「Create milestone」をクリック

## 6. ユーザーストーリーの登録

### ユーザーストーリーIssueのテンプレート

```markdown
## ユーザーストーリー
[説明文]

## 受け入れ基準
- [ ] AC-XXX-X: [受け入れ基準名]
  - [詳細ドキュメント](../requirements/acceptance-criteria.md)を参照

## 優先度
- [ ] P0（最優先）
- [ ] P1（次優先）
- [ ] P2（後回し）

## 開発フェーズ
- [ ] Phase 2（基盤構築）
- [ ] Phase 3（MVP - コア機能）
- [ ] Phase 4（拡張機能）
- [ ] Phase 5（高度な機能）
- [ ] Phase 6（最適化・運用準備）

## 見積り
[ポイント数]ポイント

## 詳細
- [詳細項目1]
- [詳細項目2]
- ...

## 考慮事項
- [考慮事項1]
- [考慮事項2]
- ...

## 関連エピック
- Epic X: [エピック名] (#[エピックIssue番号])

## 関連ドキュメント
- [プロダクトバックログ](../requirements/product-backlog.md)
- [受け入れ基準](../requirements/acceptance-criteria.md)
```

### ユーザーストーリーの登録手順

1. リポジトリの「Issues」タブをクリック
2. 「New issue」をクリック
3. 上記テンプレートを使用してIssueを作成
4. 適切なラベルを追加
5. 関連するエピックIssueにリンク
6. マイルストーンを設定
7. カスタムフィールドを設定
8. 「Submit new issue」をクリック

---

## 7. GitHub Projectsへの追加

### 手順

1. 作成したIssueをGitHub Projectsに追加
2. プロジェクトのビューを設定：
   - **ボードビュー**: ステータス別に表示（未着手/進行中/レビュー中/完了）
   - **テーブルビュー**: 一覧表示（優先度、見積り、フェーズでソート可能）
3. フィルターを設定：
   - エピック別
   - 優先度別
   - 開発フェーズ別

---

## 8. ワークフローの設定

### 推奨ワークフロー

1. **Issue作成時**
   - ステータス: `未着手`
   - ラベル: エピック、優先度、フェーズを設定

2. **開発開始時**
   - ステータス: `進行中`
   - 担当者をアサイン

3. **レビュー準備完了時**
   - ステータス: `レビュー中`
   - PRを作成してIssueにリンク

4. **完了時**
   - ステータス: `完了`
   - PRをマージ
   - Issueをクローズ

---

## 9. 自動化の設定（オプション）

### GitHub Actionsの活用

以下のような自動化が可能です：

- Issue作成時の自動ラベル付け
- PRマージ時のIssue自動クローズ
- マイルストーン進捗の自動更新

- 週次でバックログの優先順位を確認
- マイルストーンの進捗を確認
- 見積りと実績の比較
- ブロッカーの特定と対応
